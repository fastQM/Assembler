# ClawdCity 测试计划（2026-02-16，周一早上）

## 目标

1. 验证 Go 服务在 Windows / Ubuntu / macOS 三系统可运行。
2. 验证两个公网/局域网 IP 下，多机并发访问同一服务可用。
3. 验证扑克核心流程：建桌、入桌、commit-reveal、下注、事件流。
4. 验证异常场景：非法动作、重复提交、跨机网络中断后恢复。

## 拓扑

- 机器 A（macOS）：建议作为主服务节点。
- 机器 B（Ubuntu）：远程客户端 + 可选服务节点。
- 机器 C（Windows）：远程客户端 + 可选服务节点。
- IP-1：主节点地址（例如机器 A）。
- IP-2：备节点地址（例如机器 B，用于切换测试）。

## 测试前准备（全部机器）

1. 安装 Go（建议 >= 1.22）。
2. 拉取同一份代码到本地路径。
3. 放通防火墙端口（建议 8080/18080）。
4. 校时（NTP 开启），避免日志时间线混乱。

## 启动命令

### 主节点（机器 A）

```bash
cd /path/to/ClawdCity
GO111MODULE=on go run ./cmd/server \
  -addr 0.0.0.0:8080 \
  -transport libp2p \
  -p2p-listen /ip4/0.0.0.0/tcp/4001
```

### 远程机器冒烟（机器 B/C）

```bash
cd /path/to/ClawdCity
GO111MODULE=on go run ./cmd/smoke -base http://IP-1:8080
```

Windows PowerShell 同样可执行 `go run`。

### 备节点（机器 B，IP-2）

```bash
cd /path/to/ClawdCity
GO111MODULE=on go run ./cmd/server \
  -addr 0.0.0.0:8080 \
  -transport libp2p \
  -p2p-listen /ip4/0.0.0.0/tcp/4001 \
  -p2p-bootstrap /ip4/IP-1/tcp/4001/p2p/PEER_ID_OF_MACHINE_A
```

## 执行顺序

### 阶段 1：单机基线（机器 A）

1. `go test ./...`，预期全部通过。
2. 启动 server，浏览器打开 `http://127.0.0.1:8080`。
3. 在 Web UI 完成：
   - 创建 session
   - `alice/bob` join
   - start_hand
   - commit/reveal（可用 `POST /api/hash` 生成 commit）
4. 验证 `view/events` 内容更新，`stream` 有实时事件。

### 阶段 2：三机接入同一 IP（IP-1）

1. B/C 机器打开 `http://IP-1:8080`。
2. 三机分别用不同玩家名加入同一 session（例如 `alice/bob/cindy`）。
3. 顺序提交动作，验证 turn 控制正确（非当前玩家动作被拒绝）。
4. 三机同时观察：
   - `GET /events` 日志一致
   - `GET /view?player_id=...` 各自底牌隔离（只见自己的 hole cards）

### 阶段 3：自动化冒烟（B/C -> IP-1）

1. 在 B 执行一次 `cmd/smoke`。
2. 在 C 连续执行 10 次 `cmd/smoke`（手动循环即可）。
3. 预期：全部返回 `smoke test passed`。

### 阶段 4：双 IP 切换（IP-1 <-> IP-2）

1. 在机器 B 启动第二个 server：`-addr 0.0.0.0:8080`，形成 IP-2。
2. 机器 C 改为请求 `http://IP-2:8080`，执行 smoke。
3. 比较 IP-1 与 IP-2 的行为一致性（同样 API 返回结构、错误码）。

### 阶段 5：异常用例

1. 重复 commit/reveal：预期报错 `already submitted`。
2. 非 turn 玩家下注：预期 `not player turn`。
3. check 在落后下注时执行：预期 `cannot check when behind current bet`。
4. 断网 30 秒后恢复（关闭/恢复客户端网络），重新拉取 `/events`，预期可继续操作。

## 通过标准（最小）

1. 三系统均能启动 server 或执行 smoke。
2. 三系统连接 IP-1 时，核心流程成功率 >= 95%。
3. 阶段 5 的错误码和错误信息符合预期。
4. 无服务崩溃、无死锁、无明显内存飙升。
